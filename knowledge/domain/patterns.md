# Domain Patterns

> **Confirmed right approaches — verified in practice, portable across tasks.** Written by LEARN, read by THINK. A pattern is not a theory or a best practice from a blog post. It is an approach this workspace used, observed to work, and chose to repeat. If you cannot name a task where it proved correct, it does not belong here.

## How to write an entry

- **ID:** `PT-N` — sequential, never re-numbered.
- **Name:** short phrase naming the approach.
- **What it is:** one sentence — the approach itself.
- **When to apply:** the triggering condition.
- **When NOT to apply:** the boundary — every pattern has one.
- **Verification signal:** how you know it worked.
- **Provenance:** `proven in: {task-slug} · {date}` — the task that confirmed this approach.
