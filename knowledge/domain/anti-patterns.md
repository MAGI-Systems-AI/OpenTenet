# Domain Anti-Patterns

> **Written by LEARN, read by THINK.** Each entry is a failure mode the workspace has actually encountered — not theory, not general advice. The AP CHECK in THINK scans this file before execution. If an AP matches the current task, cite it by ID.

## How to write an entry

- **ID:** `AP-N` — sequential, never re-numbered.
- **Name:** short phrase naming the failure mode.
- **What happened:** one sentence — the context and the bad outcome.
- **Why it fails:** the root cause, one sentence.
- **Mitigation:** what to do instead, named specifically enough to act on.
- **Provenance:** `earned from: {task-slug} · {date}` — the task that produced this entry.
